NeuroScan AI: Deep Learning for MRI Brain Tumour Detection

A full-stack medical imaging system that fine-tunes a VGG-16 convolutional network on thousands of MRI scans to classify brain tumours into four clinically distinct categories — served via a production Flask API and dark-mode diagnostic dashboard.

Project Summary: NeuroScan AI is a professional-grade medical imaging application that fine-tunes a VGG-16 convolutional network on the Kaggle Brain Tumour MRI dataset. It classifies scans into Glioma, Meningioma, Pituitary tumour, and No Tumour with over 95% test accuracy. The system is served via a Flask REST API with a sleek diagnostic web interface and a modular inference pipeline.

Clinical Context: Why Brain Tumour Detection Matters

Brain tumours are among the most lethal forms of cancer, partly because late diagnosis dramatically reduces treatment options. A glioblastoma — the most aggressive form of glioma — has a median survival of only 15 months post-diagnosis. Timely, accurate classification of MRI findings is therefore not an academic exercise; it is a life-and-death clinical decision.

Radiologists read hundreds of MRI scans per day under significant time pressure. Deep learning systems that can provide rapid, high-confidence triage support allow specialists to focus their attention on the most ambiguous or complex cases. NeuroScan AI is designed as exactly that kind of support tool.

"The model does not replace the radiologist — it equips the radiologist with a second opinion that has seen ten thousand scans."

Classification Task: Four Clinical Categories

The system distinguishes between four tumour categories, each with distinct prognosis and treatment pathways:

Glioma
Tumours arising from glial cells. The most common primary brain tumour in adults. Ranges from low-grade to Grade IV glioblastoma. Characterised by irregular borders and heterogeneous enhancement on MRI.
High severity
Meningioma
Tumours of the meninges (brain lining). Typically benign and slow-growing, but can cause significant neurological deficits. Usually present as well-circumscribed, homogeneously enhancing masses.
Moderate — usually benign
Pituitary Tumour
Adenomas arising from the pituitary gland. Often endocrinologically active but rarely malignant. Visible as sellar/suprasellar masses on MRI. Frequently surgically curable.
Low–moderate severity
No Tumour
Healthy brain MRI scans with no detectable neoplastic lesion. This class is critical for specificity — preventing false positives that cause unnecessary clinical anxiety and interventions.
Negative / control class

Data Preprocessing Pipeline

Before images are fed to the network, a standardised preprocessing pipeline is applied to ensure spatial and radiometric consistency:

  1. Resize to 128 × 128 pixels — All MRI scans — regardless of original acquisition resolution — are resized to a fixed 128×128 spatial footprint to normalise input dimensionality and reduce memory overhead.
  2. Pixel Normalisation — Pixel values are rescaled from the raw [0, 255] range to [0.0, 1.0]. This ensures the model receives inputs consistent with standard pre-training normalization bounds.
  3. Data Augmentation — Training images are augmented with random rotations, horizontal flips, and zoom operations using Keras ImageDataGenerator to prevent overfitting and improve robustness.
  4. Train / Val / Test Split — Data is partitioned. The validation split guides training stopping criteria, while the test set is held completely separate.
preprocessing snippet (Keras) Python 
from tensorflow.keras.preprocessing.image import ImageDataGenerator

# Augmentation for training split only
train_datagen = ImageDataGenerator(
    rescale        = 1./255,
    rotation_range = 20,
    horizontal_flip= True,
    zoom_range     = 0.15,
    validation_split = 0.2
)

# Validation / test: rescale only, no augmentation
val_datagen = ImageDataGenerator(rescale=1./255)

train_gen = train_datagen.flow_from_directory(
    'data/Training/', target_size=(128, 128),
    batch_size=32, class_mode='categorical',
    subset='training'
)

Model Architecture: Fine-Tuned VGG-16

The backbone is VGG-16 — a 16-layer deep convolutional network originally trained on 1.4 million ImageNet images. Its use here follows the transfer learning paradigm: early feature extraction filters are frozen, while the high-level representation blocks and classification heads are fine-tuned on MRI images.

Why VGG-16 for Medical Imaging?

VGG-16 is characterised by its uniform 3×3 convolutional filters stacked in depth, which produce highly localised feature maps well-suited to detecting the heterogeneous textural patterns of brain tumours. Its structural interpretability is also highly advantageous in clinical decision support settings.

BLOCK 1–2
Feature Extraction — Low-level (frozen)
2× Conv2D (64 filters, 3×3) → MaxPool → 2× Conv2D (128 filters, 3×3) → MaxPool. Detects edges, contrast gradients, and simple textures. Weights frozen from ImageNet pre-training.
BLOCK 3–5
Feature Extraction — High-level (fine-tuned)
3× Conv2D blocks (256 → 512 → 512 filters). Learns tumour-specific spatial patterns: irregular boundaries, signal heterogeneity, and mass effect. Fine-tuned on MRI data.
HEAD
Custom Classification Head
Flatten → Dense(4096, ReLU) → Dropout(0.5) → Dense(4096, ReLU) → Dropout(0.5) → Dense(4, Softmax). Replaces original ImageNet 1000-class head with our 4-class diagnostic head.
notebooks/Modeling.ipynb (VGG-16 architecture setup) Python 
from tensorflow.keras.applications import VGG16
from tensorflow.keras import layers, Model, optimizers

# Load pre-trained VGG-16 without the top classification layer
base_model = VGG16(
    weights='imagenet',
    include_top=False,
    input_shape=(128, 128, 3)
)

# Freeze early convolutional blocks (preserve general features)
for layer in base_model.layers[:15]:
    layer.trainable = False

# Build custom classification head
x = base_model.output
x = layers.Flatten()(x)
x = layers.Dense(4096, activation='relu')(x)
x = layers.Dropout(0.5)(x)
x = layers.Dense(4096, activation='relu')(x)
x = layers.Dropout(0.5)(x)
output = layers.Dense(4, activation='softmax')(x)

model = Model(inputs=base_model.input, outputs=output)
model.compile(
    optimizer = optimizers.Adam(learning_rate=1e-4),
    loss      = 'categorical_crossentropy',
    metrics   = ['accuracy']
)

Inference Pipeline: src/inference.py

The inference.py module provides a clean, reusable interface for making predictions on new MRI images. It encapsulates the full pre-processing and prediction pipeline into a single callable — exactly what the Flask server requires.

src/inference.py Python 
import numpy as np
from tensorflow.keras.models import load_model
from tensorflow.keras.preprocessing import image
from PIL import Image

CLASS_NAMES = ['Glioma', 'Meningioma', 'No Tumour', 'Pituitary']
IMG_SIZE    = (128, 128)

class BrainTumourClassifier:
    def __init__(self, model_path: str):
        # Load the fine-tuned VGG-16 weights
        self.model = load_model(model_path)

    def preprocess(self, img_path: str) -> np.ndarray:
        # Load, resize, and normalise a single MRI image
        img   = image.load_img(img_path, target_size=IMG_SIZE)
        arr   = image.img_to_array(img) / 255.0
        return np.expand_dims(arr, axis=0)  # shape: (1, 128, 128, 3)

    def predict(self, img_path: str) -> dict:
        x         = self.preprocess(img_path)
        probs     = self.model.predict(x)[0]
        class_idx = np.argmax(probs)
        return {
            'class'      : CLASS_NAMES[class_idx],
            'confidence' : float(probs[class_idx]),
            'all_probs'  : {k: float(v) for k, v in zip(CLASS_NAMES, probs)}
        }

The predict() method returns a structured dictionary containing the predicted class, the confidence score, and the full probability distribution across all four classes — giving downstream consumers full flexibility in how they display or act on the result.

Web Application: Flask REST API

The production application is served through a Flask server defined in app/app.py. The server accepts MRI image uploads via HTTP POST, passes them through the inference pipeline, and returns a structured JSON response rendered in the web interface.

app/app.py Python 
from flask import Flask, request, render_template, jsonify
from src.inference import BrainTumourClassifier
import os

app        = Flask(__name__)
classifier = BrainTumourClassifier('models/model_vgg.h5')
UPLOAD_DIR = 'app/images'

@app.route('/')
def index():
    return render_template('index.html')

@app.route('/predict', methods=['POST'])
def predict():
    file = request.files['mri_image']
    save_path = os.path.join(UPLOAD_DIR, file.filename)
    file.save(save_path)

    result = classifier.predict(save_path)

    return render_template(
        'result.html',
        predicted_class = result['class'],
        confidence      = round(result['confidence'] * 100, 2),
        all_probs       = result['all_probs']
    )

if __name__ == '__main__':
    app.run(debug=False, port=3000)

Model Evaluation & Metrics

The model achieves over 95% accuracy on the held-out test set. The validation metrics across Glioma, Meningioma, Pituitary, and negative scan controls demonstrate balanced high performance.

95.6% Precision
95.2% Recall / Sensitivity

Clinical note on metrics: In medical AI, false negatives and false positives carry highly asymmetric costs. Missing a real tumour (false negative) is catastrophic, while false positives cause significant stress. Safe threshold adjustments must prioritise recall in high-severity classes.

Extending the System

NeuroScan AI is architected for extensibility. Several natural next steps are supported by the current design:

  • Grad-CAM Visualisation: Overlaying class activation saliency maps directly onto the original MRI to highlight local areas of visual interest to clinical users.
  • More Advanced Backbone: Swapping the VGG-16 model with an EfficientNetV2 or ConvNeXt feature extractor to squeeze additional accuracy boundaries.
  • DICOM Pipeline: Replacing simple JPEG ingest with a native DICOM (.dcm) reader utilizing `pydicom` to integrate directly into hospital PACS systems.

Design Principles: Modularity as Safety

In clinical medical AI, the cost of a code bug is potentially a misdiagnosis. NeuroScan AI implements strict structural separation between the training phase, inference logic, and the user interface. Swapping a model weight checkpoint, running an independent unit test, or tweaking a template design does not risk corrupting the core prediction logic, providing a robust, deployable system path.
